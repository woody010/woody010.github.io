    存储控制器 0x48000000

    一共有13个存储控制器。

    1. 位宽和等待控制寄存器 BWSCON(BUS WIDTH & WAIT CONTROL REGISTER) 每4位控制一个BANK。 STx：启动/禁止SDRAM的数据掩码引脚，对于SDRAM，此位为0；对于SRAM，此位为1。 WSx：是否使用存储器的WAIT信号，通常设为0。 DWx：使用两位来设置BANK的位宽，00为8位；01为16位；10为32为；11保留。 比较特殊的是BANK0，它没有ST0和WS0，DW0为只读，由硬件跳线觉得，只有16、32两种位宽。 例如：BWSCON的值为0x22011110。
    2. BANK控制寄存器 BANKCONx(BANK CONTROL REGISTER，x为0-5) 外接设备的访问时序，使用默认的0x0700即可。
    3. BANK控制寄存器 BANKCONx(BANK CONTROL REGISTER，x为6-7) 在8个BANK中，只有BANK6和BANK7可以外接SRAM或SDRAM。 MT([16:15])：用于设置本BANK外接的是ROM/SRAM还是SDRAM。00为SRAM，11为SDRAM。 当MT=00时，此寄存器与BANKCON0 ~ BANKCON5类似；当MT=11时，此寄存器其他值如下： Trcd([3:2])：RAS to CAS delay，设为推荐值01。 SCAN([1:0])：SDRAM的列地址位数。00表示8位，01表示9位，10表示10位。 例如：BANKCON6/7可设为0x00018005。
    4. 刷新控制寄存器REFRESH(REFRESH CONTROL REGISTER)：设为0x008C0000 + R_CNT REFEN([23])：0为禁止SDRAM的刷新功能，1为开启SDRAM的刷新功能。 TREFMD([22])：SDRAM的刷新模式，0为CBR/Auto Refresh，1为Self Refresh（一般在系统休眠时使用）。 Trp([21:20])：设为0即可。 Tsrc([19:18])：设为11即可。 Refresh Counter([10:0])：即上述的R_CNT。 R_CNT = 2^11 + 1 - SDRAM时钟频率(MHz) * SDRAM刷新周期(uS) 例如：未使用PLL时，R_CNT = 2^11 + 1 - 12 * 64/8192 = 1955 = 0x07A3
    5. BANKSIZE寄存器（BANKSIZE REGISTER） BURST_EN([7])：0为ARM核禁止突发传输，1为ARM核支持突发传输。 SCKE_EN([5])：0为不使用SCKE信号令SDRAM进入省电模式，1为使用SCKE信号。 SCLK_EN([4])：0为时刻发出SCLK信号，1为仅在访问SDRAM时才发出SCLK信号（推荐）。 BK76MAP([2:0])：设置BANK6/7的大小。 010 = 128M/128M， 001 = 64M/64M， 000 = 32M/32M 111 = 16M/16M，110 = 8M/8M，101 = 4M/4M， 100 = 2M/2M 例如：BANK6外接64M的SDRAM，BANKSIZE可设置为0XB1。
    6. SDRAM模式设置寄存器MRSRBx(SDRAM MODE REGISTER SET REGISTER，x为6~7) CL([6:4])：SDRAM时序的时间参数。 000 = 1 clock， 010 = 2 clocks， 011 = 3 clocks 例如：MRSRB6/7可设置为0x30。

    MMU

    NAND Flash控制器(2440) 0x4E000000

    1. NFCONF NAND Flash控制寄存器 设置NAND Flash的时序参数TACLS、TWRPH0、TWRPH1，设置数据位宽。
    2. NFCONT：S3C2410没有这个寄存器 使能或禁止NAND Flash控制器，使能或禁止控制引脚信号nFCE、初始化ECC。
    3. NFCMD ：NAND Flash命令寄存器
    4. NFADDR ：NAND Flash地址寄存器
    5. NFDATA ：NAND Flash数据寄存器
    6. NFSTAT ：NAND Flash状态寄存器 只用到位0，0：busy，1：ready。

    中断

    中断模式异常向量地址：0x00000018 快速中断异常向量地址：0x0000001C

    中断寄存器

    1. SUBSRCPND (SUB SOURCE PENDING) SUBSRCPND寄存器用来标识INT_RXD0、INT_TXD0等中断（S3C2440有15个这类中断，2410有11个）。 某位为1表示发生中断。 清除中断时，往SUBSRCPND某位写1即可令此位为0，写入0无效果。
    2. INTSUBMSK (INTERRUPT SUB MASK) 某位为1表示对应的中断被屏蔽
    3. SRCPND(SOURCE PENDING)
    4. INTMSK(INTERRUPT MASK) 只能屏蔽IRQ中断，不能屏蔽FIQ中断
    5. INTMOD(INTERRUPT MODE) 某位为1表示FIQ，只能有一位被设为1
    6. PRIORITY 中断优先级的判断通过7个仲裁器来完成，包括6各一级仲裁器和1个二级仲裁器。 PRIORITY寄存器使用3位来控制每个仲裁器，ARB_MODE([0:6])和ARB_SEL(7:20)
    7. INTPND(INTERRUPT PENDING) 经过中断优先级仲裁后，优先级最高的中断在INTPND寄存器被置位，同一时间只能有一位被置1。 清楚中断时，往这个位写1。
    8. INTOFFSET(INTERRUPT OFFSET) 表示INTPND寄存器中哪位被置1了。 在清除SRCPND、INTPND寄存器时，INTOFFSET寄存器被自动清楚。

    系统时钟和定时器

    PLL时钟控制逻辑给整个芯片提供三种时钟：FCLK用于CPU核；HCLK用于AHB；PCLK用于APB。 AHB(Advanced High Performance Bus)主要用于高性能模块，如CPU、DMA和DSP等。 APB(Advanced Peripheral Bus)用于低带宽的周边外设，如UART、I2C。 UPLL专用于USB设备，MPLL用于设置FCLK、HCLK、PLCK。

    1. LOCKTIME寄存器(LOCK TIME COUNT) MPLL启动后需要等待一段时间（Lock Time），使得其输出稳定。在这段时间内，FCLK停振，CPU停止工作。 S3C2410中，位[23:12]用于UPLL，位[11:0]用于MPLL，默认值为0x00FFFFFF S3C2440中，位[31:16]用于UPLL，位[15:0]用于MPLL，默认值为0xFFFFFFFF
    2. MPLLCON(Main PLL Control) 位[19:12]为MDIV，位[9:4]为PDIV，位[1:0]为SDIV 对于S3C2410，MPLL(FCLK) = (mFin)/(p2^s) 对于S3C2440，MPLL(FCLK) = (2mFin)/(p*2^s) 其中：m = MDIV + 8, p = PDIV + 2, s = SDIV, Fin为晶振频率12M。
    3. CLKDIVN(CLOCK DIVIDER CONTROL) 用于设置FCLK、HCLK、PCLK S3C2440还需要设置CAMDIVN寄存器。

    PWM定时器 S3C2410和S3C2440的定时器部件完全一样，共有5个16位定时器。 定时器部件的时钟源为PCLK，首先通过两个8位的预分频器降低频率，然后进入第二级分频器，它们输出5种频率的时钟：2分频、4分频、8分频、16分频或者外部时钟TCLK0/TCLK1。

    1. TCFG0(TIMER CONFIGURATION) 位[7:0]、位[15:8]分别被用于控制预分频器0、1。 经过预分频器出来的时钟频率为：PCLK/(Prescaler value + 1)
    2. TCFG1 分频器选择，定时器的工作频率 = PCLK/(Prescaler value + 1)/(divider value) 每4位设置一个定时器，0000 = 1/2, 0001 = 1/4, 0010 = 1/8, 0011 = 1/16, 01xx = External TCLK0/1
    3. TCNTBn/TCMPBn(COUNT BUFFER REGISTER & COMPARE BUFFER REGISTER) 只用到位[15:0]，TCNTBn中保存定时器的初始计数值，TCMPBn中保存比较值。 没有TCMPB4，因为定时器4没有输出引脚。
    4. TCNTOn(COUNT OBSERVATION) 通过读取TCNTOn可以得知内部的TCNTn
    5. TCON(TIMER CONTROL) [3:0],[11:8],[15:12],[19:16],[22:20] 位0：开启停止 0停止定时器，1开启定时器 位1：手动更新 0无用，1将TCNTBn/TCMPBn的值装入内部的TCNTn、TCMPn 位2：输出反转 0TOUT不反转，1反转 位3：自动加载 0不自动加载，1在定时器计数达到0时，将TCNTBn/TCMPBn自动装入 第一次使用定时器，要设置手动更新为1，下一次如还要设置这一位，需先清0.

    WATCHDOG定时器 WATCHDOG定时器的8位预分频器将PCLK分频后，被再次分频得到4种频率：16分频、32分频、64分频、128分频。

    1. WTCON(WATCHDOG TIMER CONTROL) [15:8] 预分频系数 [5] 定时器启动 0停止，1启动 [4:3] 时钟选择 00 = 1/16, 01 = 1/32, 10 = 1/64, 11 = 1/128 ?[1] 中断使能 0禁止中断，1使能中断 [0] WATCHDOG功能，当定时器达到0时，0不输出复位信号，1输出复位信号
    2. WTDAT(WATCHDOG TIMER DATA) WTDAT被用来决定WATCHDOG的超时周期，在定时器启动后，当计数达到0时，WTDAT的值会自动传入WTCNT。
    3. WTCNT(WATCHDOG TIMER COUNT) 初始计数值

    LCD

    TFT LCD时序信号 一帧数据的时序：

    1. VSYNC 表示一帧的开始。
    2. VSPW 表示VSYNC信号的脉冲宽度为（VSPW+1）个HSYNC信号周期。
    3. VBPD VSYNC信号脉冲后，还要经过（VBPD+1）个HSYNC信号周期，有效的行数据才出现。（上边框）
    4. LINEVAL+1行有效数据
    5. VFPD+1个无效的行。（下边框） 一行数据的时序：
    6. HSYNC 表示一行数据的开始
    7. HSPW 表示HSYNC信号的脉冲宽度为（HSPW+1）个VCLK信号周期。
    8. HBPD HSYNC信号脉冲后，还要经过（HBPD+1）个VCLK信号周期，有效的像素数据才出现。（左边框）
    9. HOZVAL+1个像素的有效数据
    10. FPD+1个无效的像素。（右边框）

    VCLK(Hz) = HCLK/[(CLKVAL+1) * 2] Frame Rate

    TFT LCD控制器中REGBANK的17个寄存器可以分为6种： LCDCON1 ~ LCDCON5 LCDSADDR1 ~ LCDSADDR3 设置帧内存的地址 TPAL 临时调色板寄存器，用于快速输出一帧单色图像 LCDINTPND, LCDSRCPND, LCDINTMSK 用于LCD中断 REDLUT, GREENLUT, BLUELUT, DITHMODE 专用于STN LCD TCONSEL 专用于SEC TFT LCD
